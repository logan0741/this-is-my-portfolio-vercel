# 🍏 애플 감성 포트폴리오 (Vercel 배포 전용)

이 폴더(`vercel-deploy`)는 전체 포트폴리오 프로젝트 중 **프론트엔드(Next.js) 부분만 완벽하게 떼어내어 독립시킨 전용 폴더**입니다. 
지저분한 백엔드 서버 설정이나 DB 세팅 없이, 오직 이 폴더 내용물만 GitHub에 올려서 Vercel에 버튼 한 번으로 즉시 호스팅할 수 있도록 설계되었습니다.

---

## 🚀 초간단 Vercel 배포 가이드

1. **GitHub에 올리기**: 이 폴더(`vercel-deploy`) 안에 있는 내용물들을 통째로 새로운 GitHub Repository를 만들어 Push 합니다.
2. **Vercel 연결**: Vercel에 로그인 후 `Add New Project`를 눌러 방금 만든 Repo를 선택(Import)합니다.
3. **환경변수 세팅 (⭐ 필수 ⭐)**: 
   배포 설정 창의 `Environment Variables` 항목에 다음 값을 기입합니다. (이 값을 넣어야 관리자 버튼이 숨겨지고 순수 열람용으로 동작합니다.)
   - Name: `NEXT_PUBLIC_IS_VERCEL`
   - Value: `true`
4. **Deploy 클릭**: 배포가 1~2분 내로 끝납니다!

---

## 🏗 전체 시리즈 아키텍처 및 개발 흐름 가이드

이 프로젝트는 나중에 무거운 통계를 돌리거나 백엔드를 직접 만지고 싶어질 때를 대비해, **총 3단계(Phase)의 확장 가능한 설계도**를 가지고 제작되었습니다. 현재 열어보신 이 폴더는 그중 `Phase 1`에 해당하는 순수 프론트엔드입니다.

### 📌 Phase 1: Vercel 정적 배포 (현재 폴더)
- **개발 목표**: 별도 서버 구축 없이 Vercel의 무료 호스팅을 이용해, 타인에게 URL만 주고 포트폴리오를 빠르게 열람하도록 만듭니다.
- **기술 스택**: Next.js App Router, Tailwind CSS, Framer Motion
- **데이터 흐름**: DB가 없는 대신 `public/data.json` 이라는 파일에 적혀있는 하드코딩 데이터를 화면에 아름답게 뿌려줍니다.
- **핵심 디자인**:
  - Apple 특유의 Glassmorphism(유리 반투명 질감) UI 적용
  - 진짜 현실의 서류첩처럼 폴더가 펼쳐지는 3D 아코디언 애니메이션

### 📌 Phase 2: 로컬 풀스택 전환 (FastAPI + MySQL)
- **개발 목표**: 포트폴리오 내용이 바뀔 때마다 코드를 고치기 번거로우니, 화면상에서 버튼을 눌러(Admin 페이지) 내용과 사진을 바로바로 등록하는 기능을 만듭니다.
- **아키텍처**: 원본 프로젝트 폴더에 있는 파이썬 백엔드(FastAPI)와 MySQL DB를 내 컴퓨터(로컬)에서 동시에 켭니다.
- **동작 방식**: `.env.local`의 모드를 `NEXT_PUBLIC_IS_VERCEL=false`로 바꾸게 되면, 프론트엔드가 하드코딩 JSON이 아닌 진짜 백엔드 API와 통신하기 시작합니다.

### 📌 Phase 3: 24시간 실서버 배포 및 스케줄러 (오라클 클라우드)
- **개발 목표**: 컴퓨터를 꺼도 관리가 되게끔 Oracle Cloud 리눅스 서버에 백엔드를 올립니다.
- **핵심 기능 (과제 자동화)**: 
  백엔드 서버에는 `APScheduler`라는 스케줄러 로직이 매일 자정 24시마다 돕니다. 등록된 깃허브 URL들에서 리포지토리의 README.md 내용을 알아서 긁어와(크롤링), 포트폴리오 상세 내용을 자동으로 업데이트합니다.

## 💻 내 컴퓨터에서 테스트 구동하기

Vercel에 올리기 전, 컴퓨터에서 미리 화면 애니메이션을 띄워보고 싶다면 터미널에서 아래 명령어를 순서대로 입력하세요:

```bash
# 1. 필요 패키지 설치
npm install

# 2. 개발 서버 시작
npm run dev
```

이후 브라우저에서 `http://localhost:3000` 으로 접속하시면 됩니다!
